## Problem
Claude generating boilerplate files wastes 60-75% of tokens. New files can be generated by cheaper AI, verified and wired by Claude.

## What Happened
Claude writes task spec, then writes the actual component from scratch. High token cost for pattern application.

## Root Cause
Token distribution mismatch. High-intelligence AI used for low-reasoning task (boilerplate generation).

## Solution / Pattern
```bash
# Claude writes spec
SPEC="Create src/pages/Dashboard.tsx.
Requirements:
1. Import getMe() from '../lib/api'
2. Show user email, full_name
3. Loading + error states"

# Kimi generates
kimi --yolo --print --final-message-only -w /project -p "$SPEC"

# Claude verifies
npm run build

# Claude wires (adds to routes, etc.) — NEVER asks Kimi to modify existing files
```

## Prevention
Rule to add to BRIEFING.md:
```
- Claude plans + verifies + wires. Kimi/Codex generates NEW files ONLY.
- Each Kimi task = ONE file (never cross-file dependencies).
- NEVER ask Kimi to modify existing files.
- Token savings: ~73% on file generation tasks.
```
